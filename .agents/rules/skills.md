# 에이전트 스킬(Skills) 작성 규칙

새로운 스킬을 등록하거나 기존 스킬을 수정할 때는 반드시 다음 구조와 원칙을 따른다.

## 1. 폴더 구조
스킬은 점진적 공개(progressive disclosure) 패턴을 따르기 위해 파일 1개가 아닌 전용 폴더 안에 구성된다.
- **생성 위치**: `.agents/skills/[스킬-고유-이름]/`
- **필수 파일**: 해당 폴더 안에 반드시 `SKILL.md` 파일로 작성한다.

## 2. 필수 YAML 프런트매터 (Frontmatter)
모든 `SKILL.md` 파일 최상단에는 아래 형식의 YAML 프런트매터가 존재해야 한다.

```yaml
---
name: [스킬 고유 식별자, 소문자 및 공백 대신 하이픈(-) 사용]
description: [스킬의 뚜렷한 목적과 언제 사용하는지 명확한 설명. 에이전트가 판단 기준이 되는 핵심 키워드 포함.]
---
```
> 주의: `description`은 3인칭 관점으로 작성하며 에이전트가 문맥을 파악하기 쉽도록 명확해야 함.

## 3. 본문 구조
본문은 일관성을 위해 아래 양식을 따른다.

- **`# [스킬 이름]`**: 스킬의 개요를 적는다.
- **`## 언제 이 스킬을 사용하는가`**: 에이전트가 이 스킬을 발동시켜야 할 조건과 타이밍을 나열한다.
- **`## 사용 방법`**: 단계별 지침, 지켜야 할 컨벤션 및 패턴을 명시한다.

## 4. 스킬 작성 모범 사례
- ** 집중된 스킬 유지 (Single Responsibility)**: 하나의 스킬은 반드시 한 가지 작업만 잘 수행하도록 분리한다. (예: 만능 스킬 지양)
- **의사결정 프로세스 포함 (Decision Trees)**: 복잡성이 있는 경우 판단 기준을 제공해 에이전트가 올바른 결정을 할 수 있게 가이드한다.
