# 투명 객체 렌더링 정렬 스킬 (Three.js Transparent Render Order)

Three.js에서 여러 개의 투명 객체 계층(별, 그리드, 지형 등)이 동일한 원점(0,0,0)을 가질 때 발생하는 블렌딩 충돌(Z-fighting 및 렌더 순서 반전)을 방지하는 작업 지침입니다.

## 언제 이 스킬을 사용하는가

- 화면을 회전하거나 카메라 극각(Pitch)이 변경될 때 겹쳐진 투명 객체들의 앞뒤 순서가 뒤집힐 때 사용하세요.
- `AdditiveBlending`이 적용된 객체가 가려져야 할 지점 위로 갑자기 노출되면서 확 밝아지거나 사라지는 현상이 발생할 때 유용합니다.
- `depthWrite: false`로 설정된 여러 개의 거대한 Mesh/Points/Line이 동일한 Scene 원점을 공유할 때 사용하세요.

## 사용 방법

에이전트는 투명 렌더링 정렬 문제를 해결할 때 다음 단계를 따르세요:

1. **상태 진단 (Diagnosis)**: 카메라 이동 시 투명도 블렌딩 객체의 선후 관계가 뒤집히는지 확인합니다. (예: 땅 아래 별이 갑자기 땅 위로 올라오면서 눈뽕 발생)
2. **원점 확인**: 문제가 되는 매니저/객체들의 중심 원점이 동일한지 확인합니다. Three.js는 투명 객체(Z-Buffer 미기록)를 카메라와의 거리에 따라 정렬하는데, 원점이 같으면 객체들의 중심 거리가 같아 정렬 우선순위를 결정하지 못해 충돌합니다.
3. **렌더링 순서 명시 (Explicit Render Order)**:
   - 각 객체의 `renderOrder` 속성을 명시적으로 할당하여 그리기 순서를 고정하세요. 값이 높을수록 **가장 마지막에 화면 맨 앞에** 그려집니다.
   - 예시 구조 (안쪽에서 바깥쪽으로 시각적으로 덮어야 하는 순서):
     - 배경/별 (`renderOrder = 0`)
     - 가이드라인/그리드 (`renderOrder = 5`)
     - 불투명/반투명 지형 덮개 (`renderOrder = 10`)
